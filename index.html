4. Robust - content must be robust enough to be interpreted across a variety of user agents including assistive technologies

There are three types of tabindex values:
* tabindex=”0” element placed in the “natural” tab order (defined by the DOM order).
* a tabindex > 0 will place the element in a manual tab order — visited in order
* a tabindex value equal to -1 will cause the element to be programmatically focusable, but not in the tab order.

* make elements focusable with tab-index=0
* tab-index = -1 is for focus management

Semantic HTML

- introduces meaning to elements
- assistive technology needs to create an alternative user interface to suit its users needs: i.e. expressing the semantics of an element (web aim 4.1.2) - name, role & value must be able to be programmatically get/set
- voice control + switch control

role, name, state & value

Screen reader gives certain information:
1. what type of element it is: role
2. name of element
3. value of the element
4. any information about the elements state
can expect to hear element’s: role name state + val
** when we use semantic HTML accessibility is handled automatically
